transverse_pass command

Syntax: 

transverse_pass id distance D increment I
  • id = integer identification number for this transverse_pass.
  • distance D = keyword/value pair; D is a floating point number for the total length covered by the transverse pass (typically the box size transverse to the current pass direction).
  • increment I = keyword/value pair; I is a floating point specifying the transverse distance between each pass. Typically smaller than the molten zone width to achieve full coverage

Examples: 

pass 1 dir X distance 100.0 speed 10.0 

transverse_pass 1 distance 65.0 increment 10

In the above example, a pass along the "spparks" X axis is defined. The transverse_pass command defines the distance increment 10 between each pass as well as implicitly defining the number of passes based upon the keyword/value pair distance 10.0. Schematic sketch below depicts example.

Description:

This command is used in the potts/additive application to specify the parameters of the transverse_pass subcomponent of a raster pattern. Multiple transverse passes can be defined in a single input file, using unique id values.

The increment I key/value pair defines the transverse distance between passes defined in the pass command. The distance D key/value pair used together with I implicitly defines number passes when used to define a rectangular_layer.

Restrictions:

This command can only be used as part of the app_style potts/additive application.

Related commands:

pass cartesian_layer pattern

Default: none
